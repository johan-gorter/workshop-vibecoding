1

Installeren van Claude Code

Claude Code is een AI-assistent die in je terminal draait, je code begrijpt, bestanden bewerkt en opdrachten uitvoert. Je hebt een Claude-abonnement (Pro, Max, Teams of Enterprise) of een Anthropic Console-account nodig.

Windows PowerShell
irm https://claude.ai/install.ps1 | iex
macOS / Linux
curl -fsSL https://claude.ai/install.sh | bash
Alternatief: WinGet
winget install Anthropic.ClaudeCode
Tip: De native installatie werkt zichzelf automatisch bij. Bij WinGet of Homebrew moet je handmatig updaten.

Claude Code is ook beschikbaar als VS Code-extensie, als desktop-app, in de browser op claude.ai/code, en voor JetBrains IDE's.

2

Je eerste sessie starten

Open een terminal in je projectmap en start Claude Code. Bij de eerste keer word je gevraagd om in te loggen.

cd pad/naar/je/project
claude

Je ziet een welkomstscherm met sessie-informatie. Vanaf hier typ je gewoon in het Nederlands (of Engels) wat je wilt doen — alsof je een collega-ontwikkelaar aanstuurt.

Goed om te weten: Claude leest automatisch de bestanden die het nodig heeft. Je hoeft niet handmatig context toe te voegen.
Voorbeeld: eerste vraag
wat doet dit project?
3

Je codebase verkennen

Gebruik Claude om snel te begrijpen hoe een project in elkaar zit. Stel vragen zoals je die aan een senior developer zou stellen.

geef een overzicht van de architectuur

welke technologieen worden gebruikt?

hoe werkt de authenticatie in dit project?

waar zit het startpunt van de applicatie?
Tip: Gebruik @bestandsnaam om een specifiek bestand direct te delen met Claude. Bijvoorbeeld: leg @src/auth.js uit

Je kunt ook afbeeldingen plakken (screenshots, wireframes, schema's) en Claude vragen deze te analyseren.

4

Code laten schrijven en aanpassen

Beschrijf wat je wilt in gewone taal. Claude zoekt de juiste bestanden, toont de voorgestelde wijzigingen, en vraagt je goedkeuring voordat het iets verandert.

voeg een contactformulier toe aan de homepage

maak een functie die e-mailadressen valideert

voeg input-validatie toe aan het registratieformulier
Belangrijk: Claude vraagt altijd toestemming voordat het bestanden wijzigt. Je behoudt volledige controle.

Voor bugs: beschrijf het probleem of plak de foutmelding erin. Claude zoekt de oorzaak en stelt een fix voor.

gebruikers krijgen een leeg scherm na het inloggen — los dit op
5

Git en versiebeheer gebruiken

Claude werkt direct met Git. Je kunt commits, branches en pull requests aanmaken via gewone opdrachten.

welke bestanden heb ik gewijzigd?

commit mijn wijzigingen met een duidelijke beschrijving

maak een nieuwe branch genaamd feature/contact-formulier

maak een pull request aan
Tip: Installeer de gh CLI (GitHub CLI) voor de beste ervaring met pull requests, issues en code reviews.
6

CLAUDE.md instellen

CLAUDE.md is een speciaal bestand in je projectmap dat Claude bij elke sessie leest. Hierin zet je projectregels, codeerstijl en werkafspraken.

Snel genereren
/init ← analyseert je project en maakt een startbestand
Voorbeeld CLAUDE.md
# Codestijl
- Gebruik ES modules (import/export), geen CommonJS
- Schrijf code-comments in het Nederlands

# Workflow
- Draai altijd tests na codewijzigingen
- Gebruik TypeScript strict mode

Plaatsing:

  • Projectroot — gedeeld met je team (check in via Git)
  • Home-map ~/.claude/CLAUDE.md — geldt voor al je projecten
  • Submappen — Claude pakt ze automatisch op wanneer relevant
Tip: Houd CLAUDE.md kort en bondig. Te lange bestanden zorgen ervoor dat Claude instructies gaat overslaan.

8

Sessies beheren

Claude onthoudt je gesprek, maar de context-ruimte is beperkt. Goed sessiebeheer maakt Claude effectiever.

CommandoWat het doet
/clearContext wissen — doe dit tussen ongerelateerde taken
/compactSamenvatten en context vrijmaken
EscClaude stoppen (context blijft bewaard)
Esc + EscTerugspoelen naar een eerder punt
claude --continueLaatste gesprek hervatten
claude --resumeKiezen uit eerdere gesprekken
/renameSessie een naam geven zodat je deze later terugvindt
Tip: Als Claude twee keer dezelfde fout maakt, gebruik dan /clear en begin met een betere, specifiekere opdracht. Dat werkt beter dan blijven corrigeren.
9

Uitbreiden met MCP, Skills en Hooks

Claude Code is uitbreidbaar met drie mechanismen:

MCP Servers — Koppel externe tools zoals Notion, Figma, Jira of je database.

claude mcp add ← voegt een MCP-server toe

Skills — Herbruikbare workflows die je team kan delen. Maak een .claude/skills/-map met instructiebestanden.

/fix-issue 1234 ← roept een custom skill aan

Hooks — Scripts die automatisch draaien bij bepaalde acties (bijv. linter na elke bewerking).

/hooks ← hooks configureren
Verschil: CLAUDE.md-instructies zijn adviserend — Claude kan ze negeren. Hooks zijn deterministisch en worden altijd uitgevoerd.
10

Geavanceerd: automatiseren en opschalen

Zodra je comfortabel bent met Claude Code kun je het inzetten als onderdeel van je workflow en CI/CD-pipeline.

Headless mode — Claude draaien zonder interactieve sessie:

claude -p "leg uit wat dit project doet"

claude -p "analyseer deze code op beveiligingsproblemen" --output-format json

Parallelle sessies — Meerdere Claude-instanties tegelijk draaien voor verschillende taken:

# Sessie A: feature bouwen
claude "implementeer de rate limiter"

# Sessie B: code reviewen
claude "review de rate limiter implementatie"

In scripts en pipelines:

cat error.log | claude -p "wat is de oorzaak van deze fout?"

git diff main --name-only | claude -p "review deze bestanden op beveiligingsproblemen"
Tip: Claude Code werkt ook vanuit de browser op claude.ai/code en via de Claude iOS-app — start een taak op je laptop en ga verder op je telefoon.
